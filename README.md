# Jyotiska (JK)

Embedded security firmware engineer at **NXP Semiconductors** — building automotive-grade
secure boot chains and Hardware Security Module (HSM) integration for Arm Cortex-M7 ECUs.

> 🔐 The boot chain is where security is either built or broken. I work on the build side.

---

## What I work on

**At NXP** — chain-of-trust bootloaders and HSM-backed cryptographic services for automotive
ECUs on Cortex-M7, threat-modelled under ISO 21434 and integrated into the AUTOSAR security
stack.

**Open source** — contributing to
[Trusted Firmware-M](https://git.trustedfirmware.org/TF-M/trusted-firmware-m.git)
and the [Zephyr RTOS](https://github.com/zephyrproject-rtos/zephyr) security subsystem.

---

## Currently building

**secure-bootloader-cortex-m33** 🔨  
Chain-of-trust bootloader on LPC55S69 (Cortex-M33) — ECDSA-P256 image verification via
mbedTLS, A/B slot OTA update logic, and rollback protection via OTP-backed version counters.

**trustzone-m-demo** 🔨  
S-world / NS-world partitioned firmware on LPC55S69 — SAU/IDAU configuration,
Non-Secure Callable (NSC) key derivation, and secure storage accessible from the NS world.
